Usage

Four sprite files are published on jsDelivr. Pick the bundle for flexibility, or a per-size file to ship the minimum bytes. Each <symbol> is keyed by USPS abbreviation; in the bundle, IDs are suffixed with the size.

Browsers don't allow <use href="https://…/states.svg#ca-md"> across origins — that's a long-standing SVG limitation, not a jsDelivr issue. The practical pattern is to fetch the sprite once and inject it into your page, then reference symbols with a local fragment: 

// Load the sprite once on page init
fetch('https://cdn.jsdelivr.net/gh/nouchee/usa-svg-icons/states.svg')
  .then(r => r.text())
  .then(svg => {
    const div = document.createElement('div');
    div.style.display = 'none';
    div.innerHTML = svg;
    document.body.prepend(div);
  });

Then reference any state by abbreviation (and size, if using the bundle):

<!-- Bundle: include the size suffix -->
<svg width="96" height="96" style="fill: #b8342f">
  <use href="#ca-md"></use>
</svg>

<!-- Per-size sprite: abbreviation only -->
<svg width="96" height="96" style="fill: #b8342f">
  <use href="#ca"></use>
</svg>
